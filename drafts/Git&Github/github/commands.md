### git clone
`git clone` 用于将远程仓库完整复制到本地，创建包含完整历史记录的可工作副本。
**基本语法**：`git clone <仓库URL> [本地目录名]`
执行时的过程：
1. 初始化阶段
- **创建目录**：在本地创建新文件夹（如未指定则使用仓库名）
- **初始化Git仓库**：执行 `git init`，创建 `.git` 文件夹结构

2. 配置远程仓库
- **添加remote**：自动添加名为 `origin` 的远程地址
- **设置上游分支**：配置本地分支与远程分支的追踪关系

3. 数据获取阶段
- **获取对象**：下载所有 commits、trees、blobs、tags 到 `.git/objects`
- **更新引用**：获取所有分支和标签信息到 `.git/refs/remotes/origin/`

4. 文件检出阶段
- **检出默认分支**：通常是 `master` 或 `main` 分支
- **创建工作目录**：将最新版本的文件写入到工作区
